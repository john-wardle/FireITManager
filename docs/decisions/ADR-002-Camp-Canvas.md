# ADR-002: Camp Canvas

## Status
Proposed

## Context
The application needs a central workspace for incident planning and operational visualization.

## Decision
Use a central canvas-based workspace for camp layout and incident planning.

## Why
- Supports future graphical workflows.
- Keeps the main workspace flexible for custom overlays and editing interactions.
- Aligns with the dockable desktop application concept.

## Alternatives considered
- Pure table/list views
- Static widgets only
- Full map-first design

## Consequences
- The canvas will become a core part of the product experience.
- Future development should treat the canvas as a major extension point.
