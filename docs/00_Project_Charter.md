# Project Charter

## Project Name
FireIT Manager - Wildland Fire IT Management System

## Purpose
FireIT Manager is a desktop application initiative designed to support incident operations by bringing camp planning, asset tracking, network oversight, and operational coordination into a single, structured workspace.

## Mission
Provide incident teams with a dependable, desktop-first platform for coordinating camp infrastructure, inventory, connectivity, support activities, and operational visibility during complex field deployments.

## Problem Statement
Wildland fire operations often require rapid coordination across logistics, IT support, and field planning. Existing workflows are frequently fragmented across spreadsheets, notes, and disconnected tools. FireIT Manager aims to reduce friction by creating a unified, extensible operating environment for incident-focused work.

## Scope
The initial release focuses on establishing a professional desktop application shell with:
- a main incident workspace
- dockable explorer and properties panels
- a central canvas for future planning and visualization
- modular architecture for future inventory, network, reporting, and administration features

## Intended Users
- Incident managers
- Logistics and support personnel
- IT and network support teams
- Operations coordinators
- Administrators responsible for maintaining incident tooling

## Long-Term Vision
Create a reliable, field-ready software platform that supports incident operations from planning through closeout, with the flexibility to grow into a richer command-and-control system over time.

## Objectives
- Improve visibility into incident-related assets and infrastructure
- Reduce manual coordination overhead
- Provide a scalable foundation for future operational modules
- Deliver a desktop experience that is professional, maintainable, and extensible

## Success Criteria
- The application launches reliably on supported desktop systems.
- The architecture remains modular and easy to extend.
- Core workflows are understandable to developers and future contributors.
- The system provides a clear path toward full incident operations support.

## Constraints and Assumptions
- The project is initially desktop-focused rather than web-based.
- The first implementation prioritizes architecture and user experience structure over full operational feature completeness.
- The system should remain suitable for future expansion into more advanced planning and reporting workflows.
